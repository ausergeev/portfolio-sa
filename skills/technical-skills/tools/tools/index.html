Техники, инструменты и технологии

Требования

  • SRS. Применял для описания требований к проектируемым сервисам. Впоследствии делал трассировку требований и Use Cases для оценки покрытия требований.
  • Use Case. Применял графическую и табличную формы представления, в том числе для описания интеграционных кейсов.
  • User Story. Применял для представления пользовательских требований. Описывал критерии и сценарии приёмки. Формировал User Story Map для новых версий продуктов.
  • UML. Применял для моделирования различных аспектов требований. Разрабатывал диаграммы последовательности, активности, состояний, компонентов. Для визуализации применял инструменты Draw.io и Plantuml.
  • BPMN. Применял для моделирования бизнес-процессов AS IS и TO BE. Использовал в основном в учебных проектах.
  • ER, DFD, словарь данных. Применял для моделирования данных и потоков данных. Обеспечивал согласованность артефактов для оценки необходимости и достаточности данных.
  • Техники выявления требований. Применял такие техники, как интервью, анализ документов, в частности ГОСТов, а также анализ продуктов-аналогов.

Интеграции

  • REST API. Проектировал REST API с учётом модели данных. Для документирования использовал Swagger.
  • Data Mapping. Применял для сопоставления данных при проектировании интеграционных сервисов, обеспечивающих преобразование данных из различных проприетарных и открытых протоколов в REST API.

UI/UX дизайн

  • Figma, Penpot. Применял для разработки раскадровок и интерактивных прототипов с целью отображения пользовательского пути.

Тестирование

  • Client. Применял Charles, Burp при тестировании клиентских desktop- и web-приложений, в том числе для перехвата и подмены трафика.
  • API. Применял Postman, Wireshark при тестировании API для формирования тестовых запросов и анализа трафика.
  • Базы данных. Применял SQL для проверки записей в базах данных PostgreSQL.
  • Техники тест-дизайна. Применял такие техники как классы эквивалентности, граничные значения, таблицы принятия решений, попарное тестирование, состояния и переходы. Разрабатывал чек-листы, тест-кейсы, наборы и планы тестирования. Хранил тестовую документацию в TestLink.

Документация

  • ЕСПД, ЕСКД. Писал программную и эксплуатационную документацию по продуктам. Для совместной разработки документации использовал MS Sharepoint, NextCloud. Для рисования диаграмм и моделей использовал Draw.io, Visio, PlantUML.
  • Базы знаний. Писал статьи для базы знаний по продуктам в DokuWiki. В личных целях применял MkDocs, Obsidian.

Управление проектами

  • Трекеры задач. Для командной работы, постановки заданий на разработку и оформления ошибок использовал Redmine, OpenProject.
  • Методологии управления. Работал в командах, где применялся подход, близкий к методу Rolling Wave Planning (набегающая волна).

Анализ данных

  • Python, Pandas, Matplotlib, Jupyter Notebook. Применял для предобработки, анализа данных и построения аналитических отчётов. Использовал в основном в учебных проектах.
  • SQL. Применял для извлечения и анализа данных из баз данных PostgreSQL. Использовал в основном в учебных проектах.

Общетехнические навыки

  • Linux. Постоянно применял в работе, в частности Astra Linux SE. Работал в командной строке, устанавливал и конфигурировал программное обеспечение.
  • Стек TCP/IP. Работал со множеством прикладных проприетарных и открытых протоколов, анализировал и перехватывал трафик.
  • Виртуализация. Применял Virtualbox в работе для развёртывания локальной тестовой среды.
  • Контейнеризация. Применял Docker в личных целях для развёртывания сервера синхронизации заметок Obsidian на собственном NAS.
